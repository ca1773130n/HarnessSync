  <files>src/adapters/gemini.py</files>
  <action>
    **Add `_cleanup_stale_subsections` method (GMN-12):**

    1. Add a new method `_cleanup_stale_subsections(self) -> int` that:
       - Reads current GEMINI.md content via `self._read_gemini_md()`
       - If empty, returns 0
       - For each section name in `["Skills", "Agents", "Commands"]`:
         - Finds `<!-- HarnessSync:{section} -->` start marker
         - Finds `<!-- End HarnessSync:{section} -->` end marker
         - If both found, removes everything between and including the markers
         - Cleans up extra whitespace/newlines at the removal point
         - Increments removed counter
       - If any sections removed, writes cleaned content back via `self._write_gemini_md(content.strip())`
       - Returns count of removed sections

    2. **Integrate cleanup into sync flow.** The cleanup should NOT be called automatically inside sync_skills/sync_agents/sync_commands (they are independent methods). Instead, add a public method `cleanup_legacy_inline_sections(self) -> int` that wraps `_cleanup_stale_subsections`.

    3. **Call cleanup from `sync_all` override.** Override `sync_all` in GeminiAdapter:
       - Call `super().sync_all(source_data)` to run all 6 sync methods
       - Check that skills, agents, and commands results all have `failed == 0`
       - If all succeeded, call `self.cleanup_legacy_inline_sections()`
       - Return results dict

    **Safety constraint:** Only run cleanup if ALL three native-format syncs (skills, agents, commands) completed without failures. If any failed, skip cleanup to avoid data loss (the inlined version is the fallback).

    **Preserve existing content:** The main managed section (`<!-- Managed by HarnessSync -->` / `<!-- End HarnessSync managed content -->`) containing rules MUST be preserved. Only the subsection markers (Skills, Agents, Commands) inside it should be removed.

    **Also clean up `_write_subsection` usage:** Since sync_skills, sync_agents, sync_commands no longer call `_write_subsection`, the method can remain for backward compatibility but document it as legacy. Do NOT remove it (other code may reference it).
  </action>
