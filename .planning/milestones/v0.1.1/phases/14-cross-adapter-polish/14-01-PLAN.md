    **2. OC-10: Add `translate_env_vars_for_opencode_headers()` to `env_translator.py`**

    Add a new function after `preserve_env_vars_for_gemini()`:

    ```python
    def translate_env_vars_for_opencode_headers(headers: dict) -> tuple[dict, list[str]]:
        """Translate ${VAR} references in MCP headers to OpenCode {env:VAR} syntax.

        OpenCode uses {env:VAR_NAME} for environment variable interpolation in headers,
        not the ${VAR_NAME} syntax used by Claude Code. This function translates header
        string values only -- do NOT apply to url, command, or environment fields.

        Args:
            headers: Dict of header name -> header value

        Returns:
            Tuple of (translated_headers, warnings)
        """
    ```

    Implementation:
    - Iterate over header key-value pairs
    - For each string value, use regex `VAR_PATTERN` (already defined in module) to find `${VAR_NAME}` and `${VAR_NAME:-default}` references
    - Replace `${VAR_NAME}` with `{env:VAR_NAME}` using `re.sub(r'\$\{([A-Z_][A-Z0-9_]*)(:-[^}]+)?\}', ...)`
    - When `${VAR:-default}` is encountered (group 2 is not None), strip the default and emit a warning: `f"Header '{key}': default value '{default}' stripped (OpenCode {{env:VAR}} does not support defaults)"`
    - Non-string values pass through unchanged
    - Return (translated_dict, warnings_list)

    AVOID:
    - Translating env vars in non-header fields (url, command, environment) -- only headers use this syntax in OpenCode
    - Using a separate regex -- reuse the existing `VAR_PATTERN` constant
    - Mutating the input dict -- build a new dict
  </action>
