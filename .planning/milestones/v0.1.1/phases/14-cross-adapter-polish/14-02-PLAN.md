    In the `for name, source_path in skills.items():` loop (line ~135), BEFORE the `create_symlink_with_fallback` call, add:

    ```python
    # Skip skills that OpenCode natively discovers from .claude/skills/
    claude_skills_dir = self.project_dir / ".claude" / "skills"
    try:
        if source_path.is_relative_to(claude_skills_dir):
            result.skipped += 1
            result.skipped_files.append(f"{name}: natively discovered by OpenCode from .claude/skills/")

    except (ValueError, TypeError):
        pass  # Not relative to claude_skills_dir, proceed with symlink
    ```

    - Plugin-provided skills (from plugin directories)
    - Any skill path NOT under `project_dir/.claude/skills/`

    AVOID:
    - Skipping ALL skills -- only skip those under `project_dir/.claude/skills/`
    - Using string comparison instead of `is_relative_to()` (Python 3.9+ is guaranteed)

  <files>src/adapters/codex.py, tests/verify_phase14_preservation.py</files>
  <action>
    The Codex adapter's `_build_config_toml()` reconstructs config.toml from only settings + MCP sections, dropping any `[agents]`, `[profiles]`, `[features]`, or other user-defined sections. Fix this by reading the raw file, extracting non-managed content, and re-emitting it.

    **1. Add `_extract_unmanaged_toml()` method to CodexAdapter:**

    ```python
    def _extract_unmanaged_toml(self, config_path: Path) -> str:
        """Extract TOML sections not managed by HarnessSync for re-emission.

        Managed content (will be regenerated):
        - Header comments (# ... HarnessSync ...)
        - sandbox_mode and approval_policy top-level keys
        - All [mcp_servers.*] sections

        Everything else is preserved verbatim.

        Args:
            config_path: Path to existing config.toml

        Returns:
            Raw TOML string of non-managed sections, or empty string
        """
    ```

    Implementation:
    - If file doesn't exist, return `''`
    - Read file as raw text
    - Process line by line, tracking state:
      - Skip lines that are HarnessSync comments (`# ... HarnessSync ...`, `# ... MCP servers ...`, `# Do not edit ...`)
      - Skip `sandbox_mode = ...` and `approval_policy = ...` top-level key lines
      - Skip `[mcp_servers` table headers and all their content (track `in_mcp_section` flag)
      - When encountering a new `[section]` header that is NOT `[mcp_servers`, set `in_mcp_section = False` and keep the line
      - Keep all other lines verbatim
    - Return the remaining lines joined, stripped of leading/trailing blank lines

    **2. Update `_build_config_toml()` signature and body:**

    Change from:
    ```python
    def _build_config_toml(self, settings_section: str, mcp_section: str) -> str:
    ```
    To:
    ```python
    def _build_config_toml(self, settings_section: str, mcp_section: str, preserved_sections: str = '') -> str:
    ```

    After the MCP section, append preserved_sections if non-empty:
    ```python
    if preserved_sections:
        lines.append('')
        lines.append(preserved_sections)
    ```

    **3. Update `_write_mcp_to_path()` to extract and pass preserved sections:**

    Before building the final TOML, call:
    ```python
    preserved = self._extract_unmanaged_toml(config_path)
    ```
    Then pass it:
    ```python
    final_toml = self._build_config_toml(settings_section, mcp_toml, preserved)
    ```

    **4. Update `sync_settings()` similarly:**

    Before building final TOML:
    ```python
    config_path = self.project_dir / ".codex" / CONFIG_TOML
    preserved = self._extract_unmanaged_toml(config_path)
    final_toml = self._build_config_toml(settings_section, mcp_section, preserved)
    ```

    **5. Create verification test `tests/verify_phase14_preservation.py`:**

    a) Creates a config.toml with `[agents]` and `[profiles]` sections
    b) Runs `_write_mcp_to_path()` to add MCP servers
    c) Verifies `[agents]` and `[profiles]` survive in the output

    e) Runs Gemini `sync_settings()` or `_write_mcp_to_settings()`
    f) Verifies `hooks` and `security` survive

    Per user preference: search the entire codebase for ALL callers of `_build_config_toml` to ensure all paths pass preserved_sections. Expected callers: `_write_mcp_to_path()` and `sync_settings()` -- both must be updated.

    AVOID:
    - Parsing non-managed sections as TOML (preserve as raw text to maintain formatting)
    - Breaking the file if no non-managed sections exist (empty string should be no-op)
    - Only fixing one call site -- both `_write_mcp_to_path()` and `sync_settings()` write config.toml
